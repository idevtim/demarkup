## Highlights

- **Pick Element mode** — new popup button lets you click any element on the page and copy just that as Markdown. Hover shows a floating green overlay with a tag label (e.g. `button.primary-btn`), click to copy, Esc to cancel.
- **Much better API documentation conversion** — pages built with Stoplight, Swagger, and similar frameworks now produce clean output.
- **Version tag in the popup now reflects the installed version** automatically (pulled from the manifest).

## Improvements

### Content cleanup
- Strip page chrome (navigation, sidebars, footers, cookie banners, try-it-out panels) in both full-page and picker modes, not just main-content mode.
- `javascript:` and `unsafe:javascript:` links are now unwrapped to plain text for all conversion modes instead of leaving broken href markdown.
- Broader Stoplight / API-doc selectors remove the left-nav TOC, the expanded JSON Schema Viewer, request-maker / try-it panels, and "powered by" branding links.

### Code blocks
- Detect code inside `<div>` / `<section>` wrappers with class names like `language-*`, `prism-code`, `shiki`, `hljs`, `sl-code-block`, `CodeMirror` — not just `<pre><code>`.
- Handle code rendered line-by-line as separate `<div>`s or `<span>`s (Stoplight, Monaco, React Syntax Highlighter). Line breaks are preserved properly instead of collapsing.
- Strip line-number gutters from fenced code by detecting sequential integers interleaved with the code.

### Tabs
- Force-reveal hidden `[role="tabpanel"]` elements before conversion, so tabbed code examples (curl / Node / Python / etc.) all come through, each labeled with its tab name.

## Technical

- New `window.__demarkupPick` entry point, injected the same way as `__demarkupConvert`.
- New `pickElement` action on the service worker; popup → background → content-script activation.
- Conversion pipeline split into `revealHiddenTabs` → `cleanContent` → `resolveUrls` → Turndown.
- `cleanContent` now has three tiers: base (scripts/inputs), chrome (nav/sidebar/try-it — always), aggressive (ads/social — main-content only).
